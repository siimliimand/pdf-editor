## Domain-Specific Rules

- **PDF-to-HTML pipeline order matters** — follow the sequence in `pdf_service.py`: save → fonts → zoom → pdftohtml → images → vectors → tables → HTML → inject fonts
- **pdftohtml is capped at 3.0x zoom** — higher zoom levels are handled by the frontend via CSS
- **Table detection requires 1.0x scale XML** — if zoom ≠ 100%, a second pdftohtml run at 1.0x is needed for accurate table detection
- **Font weight detection** uses name heuristics (e.g., "Bold" → 700, "Light" → 300) — see `FontEmbedder.get_font_weight_from_name()`
- **Image extraction uses render-crop** — renders the visible page area at native resolution for quality
- **Vector parsing** uses pdfminer (not PyMuPDF) for structured vector element data
- **Tiptap custom extensions** must preserve `class`, `style`, and `id` attributes for PDF layout fidelity
- **Backend URL is hardcoded** (`localhost:8085`) in `extension/src/services/pdfService.ts` — no config mechanism
